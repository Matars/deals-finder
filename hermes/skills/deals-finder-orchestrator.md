# Deals Finder Orchestrator

Orchestrate the deals-finder project: setup, scraper generation, and self-healing.

## Project Structure

```
deals-finder/
├── config.yaml           # user config (vendors, categories, location)
├── setup.py              # interactive setup wizard
├── scrape.py             # runner — loads scrapers, runs them, outputs JSON
├── scrapers/
│   ├── base.py           # ScrapeResult interface + dynamic loader
│   ├── blocket.py        # vendor scrapers (generated by hermes)
│   ├── tradera.py
│   └── ...
├── enrichment/           # price estimation module
├── docs/                 # GitHub Pages output
│   ├── latest.json
│   ├── index.html
│   └── .scrape_errors.json  # error log for self-healing
└── hermes/
    └── skills/
        └── deals-finder-orchestrator.md  # this file
```

## Scraper Interface

Every scraper file in `scrapers/<vendor>.py` must export a `scrape(query, category)` function:

```python
import sys, os
sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.abspath(__file__))))
from scrapers.base import ScrapeResult

def scrape(query: str, category: str) -> ScrapeResult:
    """
    Returns ScrapeResult with:
      - items: list[dict] — each with title, url, price, category, source
      - errors: list[str] — error messages if scraping failed
    """
    # ... scraping logic ...
    return ScrapeResult(items=[...], vendor="vendor_name", query=query)
```

## Generating a New Scraper

When the user asks to add a vendor:

1. Get the vendor's search URL from config or user input
2. Use `browser_navigate` to visit a sample search page (e.g., with `?q=gpu`)
3. Use `browser_snapshot` or `browser_console` to analyze the DOM structure
4. Identify: listing container elements, title selectors, URL patterns, price extraction
5. Write `scrapers/<vendor>.py` following the interface above
6. Use `browser-harness` `http_get` for fast HTTP scraping (no browser needed at runtime)
7. Test with: `python scrape.py --vendor <name>`

### DOM Analysis Pattern

```javascript
// Use browser_console to explore the DOM
// 1. Find listing containers
document.querySelectorAll('article, .item, .listing, [class*="product"]').length

// 2. Extract sample data from first few containers
const items = [];
document.querySelectorAll('SELECTOR').forEach(el => {
  const title = el.querySelector('h2, h3, .title')?.textContent?.trim();
  const link = el.querySelector('a')?.href;
  const priceText = el.textContent.match(/(\d[\d\s]*)\s*(kr|€|\$)/);
  const price = priceText ? parseInt(priceText[1].replace(/\s/g, '')) : null;
  if (title && link) items.push({title, url: link, price});
});
JSON.stringify(items.slice(0, 5), null, 2);
```

### Key Decisions

- **Server-side rendered (SSR)**: Use `http_get` + regex. Fast, no browser needed. (Blocket, eBay)
- **Client-side rendered (CSR)**: Need `browser-harness` `js()` or find API. (Tradera, some Vinted)
- **Bot protection**: Note in scraper docstring, suggest fallback strategy

## Self-Healing Flow

When a scraper breaks:

1. `scrape.py` writes errors to `docs/.scrape_errors.json`
2. User runs `python setup.py --heal` (or cron detects failures)
3. The heal script generates `.hermes_heal_prompt.md` with:
   - Which scrapers failed
   - Error messages (DOM selector mismatches, empty results, etc.)
   - Current scraper code
4. Hermes agent receives the prompt and:
   a. Reads the broken scraper file
   b. Navigates to the vendor site with `browser_navigate`
   c. Analyzes the current DOM structure
   d. Compares with what the scraper expects
   e. Patches the scraper with corrected selectors
   f. Tests with `python scrape.py --vendor <name>`
5. If fix succeeds, deletes `.scrape_errors.json`

### Self-Healing Detection

Scrapers should raise clear errors when DOM changes:

```python
# Good: specific error about what changed
if not articles:
    raise ValueError(f"No <article> elements found — DOM may have changed. "
                     f"Page length: {len(html)} bytes. "
                     f"Contains 'search': {'search' in html.lower()}")

# Bad: silent failure
if not articles:
    return ScrapeResult(items=[])  # user won't know it's broken
```

## Enrichment Integration

After scraping, enrich items with price estimates:

```bash
python scraper/enrich_json.py docs/latest.json
```

This adds `original_price`, `discount_percent`, `original_price_is_estimated` fields.

## Publishing

After scraping + enrichment:

1. Update `docs/archive/` with daily snapshots
2. Git commit + push to GitHub Pages
3. Post compact Discord notification

## Cron Integration

Set up as a Hermes cron job:

```
Schedule: 0 9 * * * (daily at 9am)
Prompt: "Run the deals-finder scraper, enrich results, publish to GitHub Pages, post Discord notification."
Skills: deals-finder-orchestrator
```

The cron job should also check for `.scrape_errors.json` and trigger self-healing if found.
